## 🏛 Architectural Patterns
- **2-tier (Client-Server):** Server hosts and manages resources delivered to the client.  
- **3-tier:** Presentation (UI), Application (business logic), and Data tiers.  
- **API Gateway:** Routes API requests from client to service; handles orchestration.  
- **Component-based Architecture:** Decomposes design into loosely coupled components.  
- **Distributed System:** Multiple services on different machines coordinate via protocols (e.g., HTTP).  
- **Event-driven:** Producers publish events, routers direct them to consumers.  
- **Microservices:** Application broken into modular services.  
- **Peer-to-Peer (P2P):** Decentralized network where nodes act as both clients and servers.  
- **Service-Oriented Architecture (SOA):** Loosely coupled services communicating over a network.  
- **Software Architecture:** Blueprint defining structures, behaviors, and interactions of a system.

## 🔄 Development Methodologies
- **Agile:** Iterative development with sprints, unit testing, and stakeholder feedback.  
- **Minimal Viable Product (MVP):** Basic feature set release for validation and feedback.  
- **Prototype:** Small-scale replica of end product for requirements gathering.  
- **Scrum Master:** Facilitates team success in Agile projects.  
- **Product Owner:** Defines vision, requirements, and ensures stakeholder value.

## 💻 Programming Concepts
- **Array:** Fixed number of elements stored sequentially.  
- **Attributes:** Object’s properties and data.  
- **Branching:** Program executes different instructions based on conditions.  
- **Boolean Expression:** Statement with values `true` or `false`.  
- **Class:** Generic version of an object with attributes and methods.  
- **Encapsulation:** Bundling data and methods, hiding internal state.  
- **Inheritance:** Subclass inherits parent properties/methods.  
- **Loop:** Sequence of instructions repeated until condition met.  
- **Function / Method / Procedure:** Reusable code performing a specific action.  
- **Conditional Statement:** `if-then` logic enabling branching.  
- **Object-Oriented Programming (OOP):** Focused on objects containing both data and methods.  
- **Pseudocode:** Human-readable logic bridging design and code.

## 🧪 Testing & Quality
- **Alpha Release:** First functioning version for select stakeholders.  
- **Beta Release:** Limited release to external stakeholders.  
- **General Availability:** Stable version for all users.  
- **Black-box Testing:** Tests without looking at source code.  
- **Functional Testing:** Tests inputs and outputs.  
- **Integration Testing:** After unit testing, components tested together.  
- **Regression Testing:** Ensures new changes don’t break existing functionality.  
- **Non-functional Testing:** Tests performance, scalability, and security.

## 🛠 Application Development Tools
- **Build Automation Utilities:** Compile and link source code into executables.  
- **Build Automation Servers:** Execute build utilities on schedule/trigger.  
- **CI/CD (Continuous Integration / Continuous Deployment):** Automates builds, tests, and deployments.  
- **Git:** Code repository for version control and branching.  
- **IDE (Integrated Development Environment):** Tools for building and managing code.  
- **Package Manager:** Installs, updates, and manages software packages.  
- **Framework:** Scaffold for building applications.  
- **Library:** Collection of reusable code.

## ☁️ Deployment Environments
- **Application Deployment Environment:** Hardware/software resources to run applications.  
- **Production Environment:** Infrastructure delivering applications to end-users.  
- **On-premises Deployment:** Organization manages system and infrastructure.  
- **Public Cloud:** Infrastructure over the internet, owned by provider.  
- **Private Cloud:** Exclusive infrastructure for one organization.  
- **Hybrid Cloud:** Combination of public and private clouds.  
- **Firewall:** Prevents unauthorized access to/from private networks.  
- **Load Balancer:** Distributes traffic across servers.  
- **Scalability:** Ability to handle dynamic load changes.

## 👥 Roles in Software Engineering
- **Backend Developer:** Manages server-side logic and data.  
- **Frontend Developer:** Manages client-side interactions and UI.  
- **Fullstack Developer:** Works across frontend and backend.  
- **QA Engineer:** Ensures product quality via testing.  
- **Project Manager:** Oversees planning, scheduling, and communication.  
- **Product Manager:** Defines product vision and roadmap.  
- **Site Reliability Engineer (SRE):** Ensures system reliability and automates processes.  
- **Information Developer:** Technical writer.  
- **Ops Engineer:** Another term for SRE.  

---

## 🚀 Career Paths & Skills
- **Chief Technology Officer (CTO):** Oversees R&D and infrastructure.  
- **Principal Engineer:** Responsible for architecture, design, and technical direction.  
- **Soft Skills:** Interpersonal and non-technical skills essential for teamwork.
